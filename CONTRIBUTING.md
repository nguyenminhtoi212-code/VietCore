# CONTRIBUTING TO VIETCORE

Thank you for your interest in contributing to the VietCore project. To maintain the high security standards and code integrity of this system, please follow the guidelines below.

## 1. CODE OF CONDUCT
- Maintain professionalism and technical accuracy in all submissions.
- Ensure all security-related discussions are handled with confidentiality.

## 2. DEVELOPMENT GUIDELINES
- Language: All source code must be written in Kotlin/Java.
- Architecture: Follow the standard Android Jetpack architecture and lifecycle management.
- Formatting: Align with the standard Android Studio code style.
- Documentation: Provide clear comments for complex security logic, especially within the OmnisSecurity and DeviceSimulator modules.

## 3. SECURITY PROTOCOLS
- Integrity: Every contribution must undergo strict integrity checks to prevent APK tampering or manifest corruption.
- Detection Layers: Ensure that any new feature does not interfere with the existing anti-root, anti-debug, and anti-emulator shield.
- Privacy: Do not implement functions that violate the user's data privacy or security layers.

## 4. PULL REQUEST PROCESS
1. Fork the repository and create your feature branch.
2. Implement your changes following the security standards mentioned above.
3. Ensure the "Self-Destruct" and "Real-time Monitoring" logic remains fully functional and stable.
4. Submit a Pull Request with a detailed description of the changes and the logic behind them.
5. Wait for the Lead Developer's review.

## 5. BUG REPORTS & SUGGESTIONS
- Use the GitHub Issues tracker to report bugs.
- Clearly describe the environment (Device, OS, Hardware) where the issue occurred.
- Provide steps to reproduce the violation or error code.
